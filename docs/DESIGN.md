# ぴよぴよことば — 設計ドキュメント

> 4〜5歳児のための、超やさしい「ことば・語彙」知育アプリ。
> ベルトコンベアで **やさい・くだもの・どうぶつ・のりもの** の絵文字がゆっくり流れてくる。
> 「**ばなな は どれ？**」の出題に、流れてくるものから **絵で** こたえる単語帳あそび。
> 正解すると ぽんっと拡大＋「ばなな！」＋読み上げ＋⭐。まちがえても ぷるぷる揺れるだけ（罰なし）。

最終更新: 2026-07-16

「ぴよぴよランド」3作目。世界観の正典は [WORLD.md](https://github.com/kerokero-1245/kerokero-1245.github.io/blob/main/docs/WORLD.md)。
第1作 [おつかいめいろ](https://github.com/kerokero-1245/otsukai-meiro)・第2作 [ぴよぴよさんすう](https://github.com/kerokero-1245/piyopiyo-sansu) と同系統。UX原則・配布方針・技術スタック・部品（大ボタン／紙吹雪／効果音合成／⭐スタンプ／親ゲート）を流用する。街（ランチャー）の施設名は「やおやさん」🥕。

## 0. 配布方針（Webファースト・確定）

**まず Web で配る。**（既存2作と同方針）Expo の web ビルド（静的サイト）を書き出し、その URL を姪っ子の端末（を持つ家族）に渡す。ブラウザで開き **ホーム画面に追加** すれば全画面で遊べる（iOS / Android）。

- **ホスティング: GitHub Pages**。`npx expo export --platform web` で `dist/` を書き出し、GitHub Actions で自動デプロイ。サブパス（`/piyopiyo-kotoba/`）配信のため、ビルド時のみ `EXPO_BASE_URL` を渡してベースパスを付ける（ローカル開発には影響させない）。
- **完全オフライン**: バックエンド・アカウント・通信・外部送信はゼロ。設定（カテゴリのオン/オフ・読み上げオン/オフ）と累計⭐だけ端末内（web は `localStorage`）に保存。消えても遊びは壊れない。
- **TTS（読み上げ）はブラウザ内蔵の `speechSynthesis`**。音声も端末内で完結（外部送信なし）。使えない環境ではひらがな表示だけで成立させる（TTS必須にしない）。

## 1. コンセプト

**文字を「読ませる」のではなく、ことばと「もの（絵）」を結びつける。** 4〜5歳で育てたいのは、耳で聞いた語 → 意味（実物イメージ）の対応づけ（語彙のネットワーク）。だから本アプリは徹頭徹尾「**ことばを聞いて／見て、それの絵をさがしてタップする**」体験にする。

- **出題 = ことば1つ** … 画面上の出題バーに「**ばなな は どれ？**」（ひらがな）＋🔊ボタン。表示と同時に読み上げ、🔊タップで何度でも聞ける。
- **えらぶ = 流れてくる絵から** … ベルトコンベアに絵文字（🍌🥕🐶🚗）が右→左にゆっくり流れる。3〜4個がループするだけで **画面外に出ても失敗にならない**（時間切れ・ゲームオーバーなし）。
- **正解 = ぽんっと祝福** … 正しいものをタップ → その絵が大きく現れ「ばなな！」＋読み上げ＋⭐スタンプ＋ちいさな紙吹雪 → つぎの問題へ。
- **まちがい = 罰なし** … タップしたものが **ぷるぷる震える** だけ。ベルトは流れ続け、何度でも挑戦できる。タイマー・スコア・ゲームオーバーは一切なし。

### この子に寄せる差別化（将来）

既存2作と同じ思想で、掛け声「ばなな！」「せいかい！」を **おじの録音声** に、辞書を姪っ子の好きなモチーフに寄せる（フェーズ2）。MVPは既定の絵文字＋ブラウザTTS＋合成音で成立させる。

### 立ち位置

| 前提 | 中身 |
|---|---|
| **プライバシー: 完全オフライン** | 設定・累計⭐は端末内のみ。TTSも端末内合成。**外部送信・アカウント・通信はゼロ**。 |
| **著作権** | 既存キャラIPは不可。素材は **絵文字／自作／AI生成／フリー素材／おじの録音** のみ。MVPは OS の絵文字をプレースホルダに使う。 |

## 3. 出題・辞書ロジック（本アプリの背骨）

### 単語辞書（`src/game/words.ts`）

1件 = `{ emoji, word(ひらがな), category }` を **約40語** 同梱。4カテゴリ×約10語。

| id | 表示（絵文字つき） | 例 |
|---|---|---|
| `yasai` | やさい🥕 | にんじん・とまと・なす・かぼちゃ… |
| `kudamono` | くだもの🍓 | ばなな・りんご・いちご・ぶどう… |
| `doubutsu` | どうぶつ🐰 | いぬ・ねこ・うさぎ・ぞう… |
| `norimono` | のりもの🚗 | くるま・ばす・でんしゃ・ひこうき… |

- **word は全語ユニーク・emoji も全語ユニーク**（重複なし）。連濁や表記ゆれを避けるため、読み上げ文字列はそのまま `word` を使う。
- カテゴリは `id` で管理し、表示ラベルと絵文字を別に持つ（おとなモードのオン/オフに使う）。

### 1問の作り方（`src/game/quiz.ts`、シード無しランダム）

1. 有効カテゴリ（おとなモードでオンのもの）から出題対象カテゴリを1つ選び、その中から **正解語** を1つ選ぶ。
2. **誤答は「別カテゴリから1語ずつ」** 選ぶ。1問の選択肢は最大4つ＝**各カテゴリから最大1語**にする。これで「ばなな🍌／にんじん🥕／いぬ🐶／くるま🚗」のように **見た目が必ずはっきり違う** 組み合わせになり、紛らわしさを構造的に排除する（🍎りんご と 🍅とまと が同じ問題に並ばない）。
3. 選択肢数 `K = min(4, 有効カテゴリ数)`。有効カテゴリが2つなら2択、3つなら3択、4つなら4択。誤答は正解カテゴリ以外の各カテゴリから1語ずつ、`K-1` 個。
4. **有効カテゴリが1つだけ**の場合のみ、例外的に同カテゴリ内から誤答を選ぶ（この時だけ見た目が似る可能性がある＝§13 仮置き）。
5. 選択肢をシャッフルしてベルトに載せる。**5問で1セット**（`generateSet(5)`）。前問と正解語が連続で被らない程度の軽い分散はするが、シード固定はしない（毎回ちがう）。

### データモデル（`src/types.ts`）

```ts
export type CategoryId = 'yasai' | 'kudamono' | 'doubutsu' | 'norimono';

export interface WordItem {
  emoji: string;      // 🍌
  word: string;       // 'ばなな'（ひらがな・読み上げにもこの文字列を使う）
  category: CategoryId;
}

export interface Question {
  answer: WordItem;   // 正解
  choices: WordItem[]; // answer を含む K 個・シャッフル済み（ベルトに流す）
}

export type Route = { name: 'title' } | { name: 'play' } | { name: 'otona' };
```

盤面座標は持たない（ベルトの並び・大きさは描画側が実測から自動レイアウト）。

## 4. 読み上げ（音声）方針

### 3段構え（同梱クリップ → speechSynthesis → 無音）

読み上げは **3段構え** で解決する。画面は入口の `src/audio/voice.ts` だけを呼び、内部で次の順に落ちる。

1. **① 同梱クリップ（tier1 / `src/audio/clips.ts`）** … 後工程で **VOICEVOX** により事前生成した音声を `assets/voice/` に同梱し、あれば最優先で再生（web は `HTMLAudioElement`、native は無音スタブ）。
2. **② speechSynthesis（tier2 / `src/audio/speech.ts`）** … ブラウザ内蔵 `window.speechSynthesis`（Web Speech API）で日本語（`ja-JP`）を合成（下記）。
3. **③ 無音（tier3）** … ①②どちらも無ければ何もしない。**ひらがな表示だけでゲームは完全に成立** する。

**クリップの対応付け**: 辞書エントリ・定型フレーズ・出題文それぞれに「クリップ基名」を対応付けられる（`voice.ts` の `wordVoice`／`PHRASE_VOICE`／`askVoice`）。基名の規約は 単語＝単語 `id`（`banana` 等）、定型句＝`p_seikai` 等、出題文＝`ask_<単語id>`。同梱表は `src/audio/voiceClips.ts` の `CLIP_URLS`（`require()` で登録）。**MVP では未同梱（`CLIP_URLS` は空）＝常に②③へフォールバック**。クリップを同梱すれば呼び出し側の変更なしに tier1 が効く。詳細と手順は `assets/voice/README.md`。

### tier2（speechSynthesis）の詳細

実装は `src/audio/speech.ts` に隔離し、web 専用（native は無音スタブ）。

- **いつ鳴らすか**: ①出題バーが新しい語を表示した瞬間、②🔊ボタンのタップ、③正解時の「ばなな！」。いずれもおとなモードの読み上げ設定が **オン** のときだけ（現状 UI は3場面とも「単語」を読む＝`sayWord`）。
- **iOS Safari 対策**: 音（クリップ・`speechSynthesis` とも）は **最初のユーザー操作より前には鳴らせない**。そこでタイトルの「あそぶ」タップ時に **1回だけアンロック** する（`warmUpVoice()`＝クリップの muted 起動＋無音発声）。以降は出題時の自動再生（タップ外）も鳴る。1ページロードにつき1回で足りる。
- **声の選択**: `getVoices()` から `lang` が `ja` で始まる音声を優先。読み上げ速度は少しゆっくり（`rate ≈ 0.9`）、`pitch` 標準。`getVoices()` が非同期な環境（Safari/Chrome）では `voiceschanged` を待って選び直す。
- **フォールバック（TTS必須にしない）**: `speechSynthesis` が無い／声が無い／設定オフ でも、**ひらがなの出題表示だけでゲームは完全に成立** する。読み上げは「あると嬉しい」補助。
- **外部送信ゼロ**: 合成は端末内。ネットワークを一切使わない。

## 5. 4歳UX原則（全画面の共通ルール／WORLD §2 準拠）

- **文字に頼らない** … 出題は読み上げ＋絵で伝わる。ひらがな短文は「読める子には嬉しい」装飾。ひらがなは最大8文字程度まで。
- **巨大タップ領域** … ベルトの絵は指1本で確実に押せる大きさ。ゆっくり流れる（速く追わせない）。
- **単発タップ中心** … ドラッグ・長押しはこども操作に使わない（親ゲートを除く）。
- **誤操作に寛容 / 罰なし** … まちがいはぷるぷる震えるだけ。タイマー・ライフ・減点・ゲームオーバーなし。**失敗状態そのものを作らない**（ベルトはループし続ける）。
- **短いセッション** … 1セット5問・数十秒〜。いつでも中断・再開。連続プレイを煽らない。
- **射幸性なし** … ガチャ・乱数報酬・ランキング・広告なし。ご褒美は「ぽんっ＋読み上げ＋ちいさな紙吹雪＋⭐スタンプ」に限定。**⭐は貯まる一方で減らない**。数字スコア・減点・失敗表示は出さない。
- **感覚にやさしく** … 強い点滅・大音量・急な動きを避ける。ベルトは静かにゆっくり。音量控えめ。
- **縦持ち優先** … 中央寄せ・最大幅制限で横画面でも崩れない。

## 6. MVP スコープ

### コアループ（これ1つを作り切る）

```
[もんだい] 有効カテゴリから正解語1つ＋別カテゴリの誤答で選択肢を作る
  → 出題バー「◯◯ は どれ？」＋🔊（表示と同時に読み上げ）
  → ベルトコンベアで選択肢の絵文字が右→左にゆっくり流れる（ループ・失敗なし）
  → タップ
     ・正解 → その絵が ぽんっと拡大＋「◯◯！」＋読み上げ → ⭐スタンプ1つ（進捗ドットが⭐に）→ ちいさな紙吹雪 → つぎの問題へ
     ・不正解 → 押した絵が ぷるぷる震える → ベルトは流れ続ける → もう一度えらべる（罰なし）
  → 5問おわったら [きょうの がんばりカード]（⭐が1つずつ揃う→紙吹雪／「あつめた ほし ⭐×N」／「もういちど」/「おうちへ」）
```

### やること（MVPに入れる）

- タイトル画面（あそぶ／親ゲート）。「あそぶ」タップで TTS をアンロック。
- もんだい画面（出題バー＋🔊＋ベルトコンベア）— 4カテゴリを混ぜて出題
- 辞書 約40語（4カテゴリ×約10語）／選択肢は正解＋別カテゴリ誤答（重複なし・紛らわしすぎない）
- 正解の ぽんっと拡大＋読み上げ／不正解の ぷるぷる（罰なし）
- 5問1セット → がんばりカード
- おとなモード（親ゲートの奥）: **カテゴリのオン/オフ・読み上げのオン/オフ・⭐リセット（2段階確認）**
- 効果音（web は Web Audio 合成、native は無音スタブ）／TTS（web の speechSynthesis、native は無音スタブ）

### やらないこと（MVP対象外 = 次段階）

- おじの声の録音・差し替え（フェーズ2）
- 辞書のモチーフ差し替え・パーソナライズ（フェーズ2）
- 文字（ひらがな）そのものの学習・なぞり書き（将来）
- 進捗レポート（親向け）／ネットワーク・バックエンド・アカウント全般

## 8. ベルトコンベア（表現の核）

- **標準 `Animated`（reanimated を足さない）** の `Animated.loop` で、選択肢の絵文字を並べたトラックを `translateX: 0 → -unitWidth` で無限にスクロールさせる。トラックは選択肢1組を必要数コピーして敷き詰め、`translateX` が1組ぶん進むたびに見た目が同一になる＝**継ぎ目のないループ**。
- **速度は一定・ゆっくり**（約 55px/秒）。速く追わせない。画面外に出た絵はループで右から戻る（**失敗状態を作らない**）。
- **タップ**: ベルト上の各コピーは `Pressable`。正解のコピーをタップすると勝ち（コピーが複数でも同一語なのでどれでも可）。誤答はその語のコピーだけ **ぷるぷる**（`shakeNonce` パターンを流用）。
- **正解演出**: ベルトを止め、中央に **リビールカード**（大きな絵文字＋「◯◯！」）を spring で ぽんっと出す＋読み上げ＋紙吹雪＋⭐。座標計算を避け、どのビューポートでも中央に確実に大きく見せる。
- **レイアウト（WORLD §2／さんすうの教訓）**: 縦を「ヘッダー＋出題バー＋ベルト」で構成し、ベルトを `flex:1` で確保。ベルトの実測サイズ（`onLayout`）から絵の1辺を算出して、可視領域を絶対にはみ出さない。コピー数はビューポート幅から動的に決める（`copies = max(2, ceil(beltW/unitWidth)+1)`）。

## 9. 技術構成

既存2作と**同一スタック**（Expo SDK 57 / React Native / TypeScript）。バックエンドなし。

| 役割 | 技術 | 用途 |
|---|---|---|
| アプリ基盤 | **Expo SDK 57 / React Native / TypeScript** | web / iOS / Android 同一コード |
| 画面遷移 | 自作の最小ステートルーター（`App.tsx`） | 文字ヘッダの無い全画面遷移（title→play→otona） |
| アニメーション | **React Native 標準 `Animated`** | ベルトの無限スクロール・リビールの ぽんっ・ぷるぷる・紙吹雪 |
| 読み上げ | **Web Speech API（`speechSynthesis`）**（web）／無音スタブ（native） | 出題・🔊・正解の読み上げ（`ja-JP`） |
| 効果音 | **Web Audio API 合成**（web）／無音スタブ（native） | タップ・正解・まちがいのやさしい音 |
| 設定保存 | `localStorage`（web） | カテゴリのオン/オフ・読み上げ・累計⭐ |

> **実装前に必ず** Expo SDK 57 公式ドキュメント（`https://docs.expo.dev/versions/v57.0.0/`）で導入手順・API を確認する（記憶で書かない）。アニメは reanimated を足さず**標準 Animated** に統一。TTS は expo 依存を増やさず **web 専用モジュール**（native 無音スタブ）で実装する。

### Web対応の注意

- **音声・読み上げの自動再生制限** … ページ読み込み直後には鳴らせない。効果音も TTS も **タップ起点**。TTS は「あそぶ」タップで1回アンロックしてから、出題時の自動読み上げを許す（§4）。
- **レイアウトが可視領域に必ず収まること（さんすうの教訓）** … ベルトを `flex:1`、実測から絵の大きさを算出。完了前に Playwright で **390×844 / 515×684 / 1280×800** を目視。背の低い画面で被りゼロを必須にする。
- **端末内保存の永続性** … `localStorage` は消え得る。設定が消えても既定（全カテゴリ・読み上げオン・⭐0）で成立するので許容。
- **画面向き** … 縦持ち前提（中央寄せ・最大幅制限）で横画面でも崩れない。

### localStorage キー（すべて `kotoba.` プレフィックス／WORLD §7 準拠）

| キー | 型 | 扱い |
|---|---|---|
| `kotoba.totalStars` | 数値文字列 | 累計⭐。1問クリアで +1（再挑戦後も +1）。減らない。**街の燃料（やおやさん）に合流**。取得失敗は 0 扱い |
| `kotoba.cats` | JSON（有効カテゴリidの配列） | おとなモードのカテゴリ オン/オフ。既定は全4カテゴリ有効。空にはできない（最低1つ残す） |
| `kotoba.tts` | `'on'` / `'off'` | 読み上げ オン/オフ。既定 `'on'` |

- 他アプリ・街のキー（`meiro.*` `sansu.*` `land.*`）は **読み書き禁止**。書いてよいのは `kotoba.*` のみ。
- 読み取りは常に try/catch ＋既定フォールバック。壊れた値でも安全に既定でレンダリング。

## 13. 決定事項・未解決

### 解決済み（この版で確定）

- **コンセプト = 記号ではなく「ことばと絵を結ぶ」**。ベルトで流れてくるものから、聞いた語の絵をタップ。
- **配布 = Web ファースト**。GitHub Pages に Actions で自動デプロイ。
- **TTS = ブラウザ `speechSynthesis`（`ja-JP`）**。「あそぶ」でアンロック、無ければひらがな表示で成立（必須にしない）。
- **選択肢 = 各カテゴリ最大1語**にして紛らわしさを構造排除（有効カテゴリ2〜4に応じて2〜4択）。
- **ごほうび = ⭐スタンプ式**（減らない）。累計 `kotoba.totalStars` は街の燃料（やおやさん）に合流。
- **おとなモード = カテゴリ オン/オフ・読み上げ オン/オフ・⭐リセット（2段階確認）**。
- **アプリ名 = ぴよぴよことば（`piyopiyo-kotoba`）**。最初からこの名前（改名なし）。

### 未解決・後で決めること（§13 仮置き）

- **有効カテゴリが1つだけの場合の誤答** … 例外的に同カテゴリ内から選ぶため、見た目が似る組み合わせが出得る（親が3カテゴリを切った時のみ）。当面は「最低1カテゴリ」を許容し、似すぎ回避の細かな除外リストは入れない。
- **辞書の語数・語選び** … 約40語で開始。長い外来語（へりこぷたー等）や連濁の扱いは運用しながら調整。
- **ベルト速度と絵の大きさ** … 55px/秒・実測ベースで開始。実機の指の当てやすさで微調整。
- **声・辞書のパーソナライズをどこまでMVPに近づけるか**（現状フェーズ2）。

## 14. ごほうび（⭐スタンプ式）— スコアではない

既存2作と同じ思想。**「点数」を付けない。** ご褒美は **⭐スタンプを“集める”** 体験に限定し、**貯まる一方で減らない**。数字スコア・減点・失敗表示・タイマーは一切出さない（§5）。

1. **1問クリア＝⭐スタンプ1つ** … 正解の ぽんっ演出のあとに⭐を1つ獲得。**再挑戦してからのクリアでも同じ⭐**。上部の進捗ドットが⭐に「ぽんっ」と変わる（`ProgressStar`）＋やさしい音（`star`）＋ちいさな紙吹雪。
2. **きょうの がんばりカード**（5問終了後）… `ClearOverlay` を流用。⭐が1つずつ順に現れる（`playCountTone`）→ 5つ揃って祝福（`Confetti`＋`clear`）。カード下に累計「あつめた ほし ⭐×N」を控えめに表示。
3. **あつめた ほし（累計）** … `kotoba.totalStars` に保存。1問クリアで `addStars(1)`。**街の燃料に合流**（やおやさん🥕が育つ）。リセットはおとなモードからのみ・2段階確認。累計は自動では減らない。
